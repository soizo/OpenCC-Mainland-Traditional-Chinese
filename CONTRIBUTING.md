# 貢獻指南

感謝你對本項目的關注！以下是提交貢獻的說明。

## 如何貢獻

### 報告問題

如果你發現轉換結果有誤，請[提交 Issue](https://github.com/soizo/OpenCC-Mainland-Traditional-Chinese/issues/new/choose) 並提供：

1. 輸入的簡體文本
2. 實際輸出的繁體文本
3. 你認爲正確的繁體寫法
4. 字形依據（如教科書、官方文件、字典等）

### 補充或修改詞庫

1. Fork 本倉庫
2. 在新分支上修改 `s2mt/SMTCharacters.txt` 或 `s2mt/SMTPhrases.txt`
3. 提交 Pull Request，說明修改原因和字形依據

## 詞庫編輯規範

### 文件格式

- 編碼：**UTF-8（無 BOM）**
- 換行：**LF（`\n`）**
- 每行格式：`簡體<TAB>繁體`
- 多個繁體候選（字級）：`簡體<TAB>繁體1 繁體2`（空格分隔，最優先的放首位）
- 空行和以 `#` 開頭的行視爲注釋

### 字形標準

本項目的繁體字形以以下資料爲準（優先級由高至低）：

1. 《通用規範漢字表》（2013）正體字形
2. 《現代漢語詞典》（商務印書館）
3. 《新華字典》（商務印書館）

不採用台灣《國字標準字體》或香港《常用字字形表》的字形。

### 詞組規則

- 在 `SMTPhrases.txt` 中，每條詞組規則都應能减少歧義（即字級規則單獨處理時可能出錯的情況）
- 避免冗餘：若字級規則已能正確轉換的詞組，不必重複添加到詞組文件

### 重複鍵

- 同一個簡體字/詞只保留一條規則（最後一條生效）
- 如需覆蓋已有規則，請刪除舊規則並添加新規則，而非重複添加
- `to_ocd2.sh` 腳本會自動注釋掉重複鍵中較早出現的條目

## 重新編譯二進制詞庫

修改 `.txt` 文件後，如需更新 `.ocd2` 二進制文件：

```bash
# 需要先安裝 OpenCC（含 opencc_dict 工具）
cd s2mt
bash to_ocd2.sh
```

或使用根目録的 Makefile：

```bash
make build
```

## Pull Request 規範

- PR 標題應簡潔說明改動內容，例如：`修正「發/髮」相關詞組的繁體字形`
- 每個 PR 應聚焦于單一問題或功能
- 若修改了 `.txt` 文件，請同時提交更新後的 `.ocd2` 文件（運行 `make build`）

## 行爲準則

請保持友善、尊重的交流氛圍。針對字形問題的討論應以字形依據爲核心，避免主觀偏好的爭議。
